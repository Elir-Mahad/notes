React best practices

React checklist

The content of this blog was taken from Mohammad Faisal. I've reproduced his best practices blog as a checklist for my personal usage.

As a self-taught developer, i've had a solid share of haphazard development which only ever produced Spaghetti code. After refactoring a few of my projects, i began to see the value of following best practices at all times. I came across a few different react best practices blogs. Below is a react best practices checklist. I plan to add more things to it as i continue my research.

Import files in order of built-in, external, internal #import

Name components using PascalCase and instances using camelCase #naming

Use camelCase for prop names or PascalCase if the prop value is a react component #naming

Don’t use underscore in method and /or function name #naming

Use JSX shorthand for passing boolean variables #jsx

Use double quotes for jsx attributes and single quotes for all other js #jsx

Always wrap jsx in parentheses #jsx

Use Fragments instead of divs when possible #jsx

Use self-closing tags for components that don't have children #jsx

Don’t add curly braces to string props #jsx

Always include alt prop in your image tags #jsx

Use ternary operators to shorten conditionals #javascript

Use object literals to improve code readability #javascript

Don’t define a function inside render #javascript

Use Implicit return instead of explicit return #javascript

Use css in javascript #javascript

Use template literals instead of concatenation #javascript

Use memo to avoid unecessary rendering #performance

Use object destructuring #performance
