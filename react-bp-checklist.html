React best practices

React checklist

React is very unopinionated about how things should be structured. This is exactly why it’s our responsibility to keep our projects clean and maintainable. Today, we will talk about some best practices to follow to improve your React application’s health. These rules are widely accepted. As such, having this knowledge is imperative.

Import files in order of built-in, external, internal

Name components using PascalCase and instances using camelCase

Use camelCase for prop names or PascalCase if the prop value is a react component

Don’t use underscore in method and /or function name

Use JSX shorthand for passing boolean variables

Use double quotes for jsx attributes and single quotes for all other js

Always wrap jsx in parentheses

Use Fragments instead of divs when possible

Use self-closing tags for components that don't have children

Don’t add curly braces to string props

Always include alt prop in your image tags

Use ternary operators to shorten conditionals

Use object literals to improve code readability

Don’t define a function inside render

Use Implicit return instead of explicit return

Use css in javascript

Use template literals instead of concatenation

Use memo to avoid unecessary rendering

Use object destructuring
