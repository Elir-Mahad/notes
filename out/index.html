How to code with other people

When i was learning how to code, i spent a lot of time coding by myself. During this time, the one question that i consistently asked myself was: Is the code that i'm writing going to work ? No other questions occupied my mind. Everytime that my code worked, i felt a sense of accomplishment; and everytime that it didn't work, i found myself haunted by the new question: Why didn't it work ?

Eventually, i started to collaborate with other developers; and here, i found myself haunted by a new set of questions:Can another person understand my coding process, by simply reading my commits ? This question indicated a major philosophical shift in my persepective. I'll dig into this shift, and its meaning, in another post. For now, i will leave you with a set of instructions that i often return to whenever i'm coding with others.

1. Pull the repo

  • If the main code is in the branch ‘master’ then

    • git pull master

    2. Work on a issues

  • Before you create a branch, write an issue outlining all the new tasks / features that you will be coding out.

    • 3. Create a branch of the master branch

  • For example, if you want to create a branch called header of the master branch, first make sure that you are on the master branch:

    • git branch

  • terminal response:

    • master

  • Then create the header branch:

    • git branch header

  • Enter the header branch:

    • git checkout header

  • Check which branch you are currenlty in:

    • git branch

  • Terminal response:

    • Switched to branch 'header'

    Finally, write the code for the header feature in this branch.

    4. Push your new branch into the master branch:

  • For example, if you just finished writing all your code for the header branch then in the terminal write:

    • git add .
    git commit -m 'Add header code'
    git push

  • If this is your first push, then you will get this response:

    • git push --set-upstream origin master

  • Copy and paste this into the terminal so that from now onwards whenever you want to push to your branch you will only type ‘git push’

    • 5. Create a pull request:

  • Make sure that the branch that you are requesting to merge your branch into, is the correct branch.
  • For example: If you are working on a branch called ‘header’ and you want to merge this header branch into the ‘master’, then on the top of the pull request it should say that you want to merge header into master.
  • In the pull request message, explain all the things that happened in this branch. Specify next steps.
  • Assign a reviewer. If your’e working by yourself then you can leave this as blank.
  • Assign an assignee. If your’e working by yourself assign this to yourself.
  • Fill out the other specifications.
  • Submit the pull request.
  • After its submitted, attach the issue name that was associated with this pull request in the add issue block on the bottom right.
  • If your’e working with a group, wait for the reviewer to review it and either accept it or reject it.
  • If your’e working by yourself, go to the bottom of the pull request and click on merge.

    • 6. Create a new branch:

  • Change the branch back to the master or whatever the branch that you're going to be merging into is called.
  • Switch to the master branch

    • git checkout master

  • Update your local repo master branch with the latest changes:

    • git pull master

  • This is an important step, especially when your’e working with multiple people who are all working on different branches.
  • For example: Imagine that you were working on the header branch, and another person was working on the menu branch.
  • Then you finished working on the header branch, submitted it, and it got merged into the master.
  • Also, the other person finished working on the menu branch, they submitted it, and it got merged into the master.
  • Now, imagine your next branch is about creating a button.
  • Before you create the button branch, you must first update your local repo with all the code that has been merged into it so far (ie, header + menu).
  • If your local repo is updated, then you should see ‘Already up to date’ in the terminal.
  • Create a new branch called button, off the updated local repo master branch:

    • git branch ‘button’

  • Enter the new button branch:

    • git checkout button
