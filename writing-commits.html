How to write commits properly

  • A commit is a set of instructions for how to go from a previous state to a new state.
  • Write a commit as if you are giving orders to:
  • a person on how to change the code.
  • the codebase to change its behaviour.
  • The preference for present-tense, imperative-style commit messages comes from Git itself.
  • Example: commit -m “Add all images to the assets folder.”
